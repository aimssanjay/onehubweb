Platforms & Categories — Dynamic Data from REST API

Project: onehubweb
Date: March 2026
Stack: React + TypeScript + Vite

1. Overview

Previously, platform names (Instagram, TikTok, YouTube) and category names were hardcoded as static arrays in multiple component files. Any change to the platform/category list required a code deployment.

This integration replaces all hardcoded data with live API calls, so the UI reflects whatever is in the database — instantly, with zero code changes needed.

3. Hooks Created

Two reusable React hooks were created in src/app/hooks/. Each hook fetches its endpoint once on mount, stores the result in state, and returns it to any component that calls it.

usePlatforms.ts

export function usePlatforms() {
  const [platforms, setPlatforms] = useState<Platform[]>([]);

  useEffect(() => {
    fetch('http://...api.../get-all-platforms')
      .then(res => res.json())
      .then(data => {
        if (data.success)
          setPlatforms(data.data.filter(p => p.is_active === 1));
      });
  }, []); // [] = run only once when component mounts

  return { platforms };
}

useCategories.ts

export function useCategories() {
  const [categories, setCategories] = useState<Category[]>([]);

  useEffect(() => {
    fetch('http://...api.../get-all-categories')
      .then(res => res.json())
      .then(data => {
        if (data.success) setCategories(data.data);
      });
  }, []);

  return { categories };
}

5. How It Works — Step by Step

1
Component mounts in the browser React renders the component. The hook (usePlatforms or useCategories) is called automatically.
2
useEffect fires once The fetch() call is made to the API. While waiting, the array is empty [].
3
API responds with data The response is parsed, filtered (e.g. only is_active = 1 platforms), and saved via setState.
4
React re-renders the component The state now holds the API data. The .map() in the JSX renders a button/option for each item.
5
UI is live and dynamic Adding or removing a platform/category in the database now automatically appears in all dropdowns and forms — no code changes needed.

7. Icon / Image Fallback Strategy

The API currently returns "icon": null for platforms and "image": null for some categories. A local mapping handles known platforms, and a fallback is used for unknowns: 

// Local mapping for known platform icons
const platformMeta = {
  Instagram: { icon: Instagram, color: 'from-purple-600 to-pink-600' },
  YouTube:   { icon: Youtube,   color: 'from-red-600 to-red-500'   },
  TikTok:    { icon: Video,     color: 'from-black to-gray-800'    },
};

// Merge API data with local meta (fallback for unknown platforms)
const socialPlatforms = apiPlatforms.map((p) => ({
  id:   p.name.toLowerCase(),
  name: p.name,
  ...(platformMeta[p.name] ?? { icon: Video, color: 'from-gray-600 to-gray-500' }),
}));

// Category image fallback
<img src={category.image ?? FALLBACK_IMAGE} />
